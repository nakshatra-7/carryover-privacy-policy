CarryOver

Privacy Policy

Effective Date: June 25, 2026

CarryOver is a Chrome extension that helps users continue long AI conversations by estimating context usage and creating compact conversation checkpoints.

1. Local-First Operation

CarryOver is designed to work locally inside your browser. The extension does not operate any external database, backend server, analytics system, or user account system.

We do not collect, store, sell, or share your conversation data on our servers. CarryOver does not upload your chats, prompts, responses, files, browsing activity, or personal information to any database owned or operated by us.

2. Access to Chat Content

CarryOver runs on supported AI chat websites, including ChatGPT and Claude, so it can read the visible conversation content on the page.

This access is used only for the extension's single purpose: estimating context usage, showing conversation status, and helping create a compressed checkpoint for continuing the conversation in a fresh chat.

All processing is performed locally in your browser by default.

3. Data Storage

CarryOver may store limited extension data locally in your browser, such as:

  • Extension settings
  • Compression mode preference
  • Temporary checkpoint text for carrying context into a new chat
  • Optional API key, if the user chooses to use API Fallback

This data is stored using Chrome extension storage. We do not receive or store this data on any server controlled by us.

4. Optional API Fallback

CarryOver includes an optional API Fallback mode. This mode is disabled unless the user chooses to enable it and provides their own OpenAI API key.

If API Fallback is enabled, conversation text needed for compression may be sent directly from the user's browser to OpenAI's API for processing. This data is not sent to us, stored by us, or accessible to us.

Users who do not enable API Fallback can use CarryOver locally without sending conversation content to any external API through the extension.

5. No Sale or Transfer of User Data

We do not sell user data.

We do not transfer user data to third parties except where required to provide a user-enabled feature, such as optional API Fallback. We do not use user data for advertising, profiling, analytics, creditworthiness, lending, or any unrelated purpose.

6. Permissions

CarryOver requests permissions only to support its core functionality:

  • Access to ChatGPT and Claude pages is used to read visible chat content and show the CarryOver panel.
  • Storage permission is used to save local extension settings and temporary checkpoint data.
  • Clipboard permission is used to copy generated checkpoints when requested by the user.
  • OpenAI API access is used only for optional API Fallback mode.

7. Data Security

Because CarryOver does not operate a backend database or server, we do not centrally store user data. Any locally stored extension data remains in the user's browser environment.

Users are responsible for managing their own browser profile, device security, and any third-party API keys they choose to enter.

8. Changes to This Policy

We may update this Privacy Policy as CarryOver evolves. Any material changes will be reflected in the latest published version of this policy.
